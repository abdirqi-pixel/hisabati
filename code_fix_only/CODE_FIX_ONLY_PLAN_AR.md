# خطة إصلاح الكود فقط - حساباتي

## الهدف
من هذه المرحلة لا نضيف ميزات جديدة، بل نصلح الأخطاء التي تظهر أثناء:
- flutter pub get
- flutter analyze
- flutter run
- flutter build apk --release
- flutter build appbundle --release

## طريقة العمل
1. شغّل الأمر.
2. انسخ نص الخطأ.
3. سجل الخطأ في شاشة إصلاحات البناء أو في ملف BUILD_ERROR_LOG_AR.md.
4. أصلح الملف المحدد.
5. أعد تشغيل الأمر.
6. لا تنتقل للخطوة التالية حتى ينجح الأمر الحالي.

## ترتيب الأولوية
1. أخطاء pubspec والحزم.
2. أخطاء imports والكلاسات المفقودة.
3. أخطاء قاعدة البيانات.
4. أخطاء الراوتر والتنقل.
5. أخطاء واجهة المستخدم.
6. أخطاء Android/iOS build.
